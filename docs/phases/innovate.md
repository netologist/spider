# Innovate
>
> Explore the solution space before committing to an approach.

**When this runs:** After Research passes Gate 1; before Plan.

**Mandatory:**

- At least 2-3 alternative approaches
- Each evaluated for: technical fit, testability score (★☆☆☆☆ — ★★★★★), trade-offs
- Rejected alternatives documented in `specs/logs/DECISIONS.md`

**Outputs:**

- 2–3 evaluated alternatives (in session / design discussion)
- Rejected alternatives (with rationale) appended to `specs/logs/DECISIONS.md`
- Selection rationale documented

**Exit gate — Gate 2 (Innovate → Plan):**

- [ ] At least 2 alternatives evaluated?
- [ ] Testability scores given?
- [ ] Selection rationale documented?
- [ ] Rejected alternatives added to DECISIONS.md?

**Constraints:** No code writing. No spec writing. Only evaluate and recommend.

Operational discipline: see the `spider-innovate` skill.
