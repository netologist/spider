# Postmortems

Incident postmortems — exceptional events, not routine. **Append-only, permanent,
never deleted.** Not blame documents: record how the system failed and what
changed so it doesn't happen again. See the SPIDER docs (Record Files).

**Why it happened:** Root cause
**Affected users:** ~N
**Duration:** X hours

- Contributing factor

**Resolution:** What was done, how it was fixed

**Actions:**

- logged to `ARCH_LOG.md`
- `_LESSONS_LEARNED.md` updated
